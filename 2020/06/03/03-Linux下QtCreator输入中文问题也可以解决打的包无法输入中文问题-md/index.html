Linux下QtCreator输入中文问题也可以解决打的包无法输入中文问题


问题:通常在linux下安装了QtCreator编译器,编写代码无法输入中文.

环境:

环境：
ubuntu16.04
搜狗输入法
Qt Creator 5.7.1

分步指南
首先需要安装 sudo apt-get install fcitx-frontend-qt5
将目录下的/usr/lib/x86_64-linux-gnu/qt5/plugins/platforminputcontexts/libfcitxplatforminputcontextplugin.so

分别拷贝到自己qt的安装目录下(目录可能不同,以自己电脑环境为准)：
~/Qt5.7.1/gcc_64/plugins/platforminputcontexts
~/Qt5.7.1/Tools/QtCreator/lib/Qt/plugins/platforminputcontexts

重启QtCeator即可

出入某些文字显示错误：

1.http://blog.sina.com.cn/s/blog_ebe5d5570102whxb.html，参考此文章进行解决。

升级5.14无法输入中文

原因主要是系统库中的libfcitxplatforminputcontextplugin.so升级了，对于5.14的版本不兼容，需要下载https://github.com/buqg/fcitx-qt5-

或者重新编译https://github.com/fcitx/fcitx-qt5替换系统库下的文件即可。
