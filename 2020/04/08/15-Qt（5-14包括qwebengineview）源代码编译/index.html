Qt（5.14包括qwebengineview）源代码编译

  2. 2. 安装：

准备：

安装vs2019
下载qt的源代码https://mirrors.tuna.tsinghua.edu.cn/qt/official_releases/qt/5.14/5.14.0/
安装qt5.14的release版本，主要用到了里面自带的jom.exe编译器
python2.7 （or later）这里一定要安装python2，不然在configure的时候提示webengine不支持此模块，如果更新了编译工具记得删除以下源码目录下的config.cache文件
perl https://www.perl.org/get.html
Ruby https://rubyinstaller.org/downloads/

安装：

cd “C:\Program Files (x86)\Microsoft Visual Studio\2019\Community\VC\Auxiliary\Build” 执行vcvarsall.bat amd64_x86
cd D:\qt-everywhere-src-5.14.0（代码目录）
configure -confirm-license -opensource -release -prefix “D:\qt-build” -qt-sqlite -qt-zlib -qt-libpng -qt-libjpeg -nomake tests -no-compile-examples -nomake examples （替换成自己相应的目录）
开始编译之前检查此命令执行结果，看是否带来影响，如跳过什么模块的编译

   3. D:\Qt\Qt5.14\Tools\QtCreator\bin\jom.exe -j8（开启8核编译，根据电脑相应调整）

   4. 前一条命令执行完 执行D:\Qt\Qt5.14\Tools\QtCreator\bin\jom.exe  install

结果:

在编译输出目录下出现相应的库

问题：

  1. fatal error C1060: “编译器的堆空间不足”——C1001错误代码
    解决办法更好配置好一点的机器，这里使用了公司65的打包机

  2. qt dxgi_format_map_autogen.cpp error
    修改文件编码为utf-8-bom

  3. NMAKE : fatal error U1077: ‘call’
    源码路径移动到最顶层
