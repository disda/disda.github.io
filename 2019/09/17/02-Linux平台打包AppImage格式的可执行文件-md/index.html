打包发布程序

Qt linux(ubuntu) 下用 linuxdeployqt 打包发布程序(AppImage可执行文件)

下载linuxdeployqt
把下载的文件放入 /usr/local/bin 内，并执行如下命令使其可以运行
chmod 777 linuxdeployqt
把qmake的路径添加到环境变量中
sudo vim /etc/profile
在文件末尾添加：
export PATH=”/home/god/Qt5.11.2/5.11.2/gcc_64/bin:$PATH”
youduqt 是Qt release出来的可执行文件，执行如下语句可生成.appImage文件同时会复制依赖的文件到当前目录
linuxdeployqt youduqt -appimage

参考：

https://www.cnblogs.com/god-of-death/p/9876777.html

https://appimage.org/

注意事项：
打appimage包的时候需要把证书放到与youduqt执行文件同一个目录下

打包发布程序时遇到的问题

问题：

ERROR: ldd outputLine: “libprotobuf.so.9 => not found”

ERROR: ldd outputLine: “libcurl.so.4.3.0 => not found”

ERROR: ldd outputLine: “libnopoll.so.0 => not found”

解决：

sudo cp libprotobuf.so.9 /lib/x86_64-linux-gnu/

sudo cp libcurl.so.4.3.0 /lib/x86_64-linux-gnu/

sudo cp libnopoll.so.0 /lib/x86_64-linux-gnu/

问题：

ERROR: ldd outputLine: “libjasper.so.1 => not found”

ERROR: for binary: “/home/xinda/Qt5.7.1/5.7/gcc_64/plugins/imageformats/libqjp2.so”

解决：

删除 /home/xinda/Qt5.7.1/5.7/gcc_64/plugins/imageformats/libqjp2.so（walter解析我们工程没有用到此文件，但编译时会引用到，他依赖的libjasper.so.1不存在，所以干脆删除。)

问题：

ERROR: ldd outputLine: “libmysqlclient_r.so.16 => not found

ERROR: for binary: “/home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers/libqsqlmysql.so”

解决:

删除 /home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers/libqsqlmysql.so

问题：

ERROR: for binary: “/home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers/libqsqlodbc.so”

解决：

删除 /home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers/libqsqlodbc.so

问题：

ERROR: ldd outputLine: “libpq.so.5 => not found”

ERROR: for binary: “/home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers/libqsqlpsql.so”

解决：

删除 /home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers/libqsqlpsql.so

问题：

sqlite3在void* sqlite3DbMallocRawNN（sqlite3 *db，u64 n）挂掉

解决：

sqlitecipher.pro中 添加 SQLITE_OMIT_LOOKASIDE，禁止后备内存分配

问题：

ERROR: Desktop file missing, creating a default one (you will probably want to edit it)
ERROR: Icon file missing, creating a default one (you will probably want to edit it)
解决：
换linuxdeployqt版本尝试中，因为不需要桌面文件，可以确定发行版5没问题
创建一个youduqt.desktop和icon文件，放到与执行文件相同的目录

问题：
qmake: could not exec ‘/usr/lib/x86_64-linux-gnu/qt4/bin/qmake’: No such file or directory
原因：
qt指向的qt安装位置不正确
解决：
cd /usr/lib/x86_64-linux-gnu/qt-default/qtchooser
sudo vim default.conf
修改并指定路径，如：/home/codebee/Qt5.6.2/5.6/gcc_64/bin

问题：
ERROR: ldd outputLine: “libprotobuf.so.12 => not found”
ERROR: for binary: “/media/xinda/_dde_data/xinda-workspace/qt-linux/src/deps/protobuf/lib/libprotoc.so.12.0.0”
ERROR: Please ensure that all libraries can be found by ldd. Aborting.
解决：
sudo cp libprotobuf.so.12 /lib/x86_64-linux-gnu/
