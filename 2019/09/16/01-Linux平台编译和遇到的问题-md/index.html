Linux平台编译及遇到的问题

一. 环境搭建步骤

  1. 安装 qt-opensource-linux-x64-5.7.1.run
  2. 安装 gcc，g++
  3. 数据库加密驱动
    编译 QtCipherSqlitePlugin 生成 libsqlitecipher.so (25上也有), 并拷贝到 /home/xinda/Qt5.7.1/5.7/gcc_64/plugins/sqldrivers中。

二. 编译时遇到的问题：

问题:
Could not start process “make” qmake_all
解决:
sudo apt-get install build-essential

问题:
/usr/bin/ld: cannot find -lGL
解决:
sudo apt-get install libgl1-mesa-dev

问题:
:-1: error: No rule to make target ‘/home/xinda/qt-linux-git/src/jgapi/../../../../../usr/lib/x86_64-linux-gnu/libssl.a’, needed by ‘libjgapi.a’. Stop.
解决:
拷贝libssl.a(\10.0.0.25\公司数据盘_software\windows\youduqt环境搭建 目录)到 /usr/lib/x86_64-linux-gnu/ 目录

问题:
:-1: error: No rule to make target ‘/home/xinda/qt-linux-git/src/youduqt/../../../../../usr/lib/x86_64-linux-gnu/libcrypto.a’, needed by ‘../output/debug/youduqt’. Stop.
解决:
拷贝libcrypto.a(\10.0.0.25\公司数据盘_software\windows\youduqt环境搭建 目录)到 /usr/lib/x86_64-linux-gnu/ 目录

问题:
:-1: error: cannot find -lssl
解决：
切换到 /usr/lib/x86_64-linux-gnu/ 目录

问题

fail to write template file

解决:

1.首先确认ca-bundle.crt是否在运行的youquqt可执行文件目录下,如果没有则添加.

2.如果已有ca-bundle.crt在相应的目录下,则需要添加查看在依赖的curl文件下查看./curl –version,查看curl是否与相应的版本的openssl进行链接.如果没有,则需要链接到相应的openssl(youduqt工程采用的是openssl_1_0_2g)进行重新编译.

问题

Error while loading shared object: libssl.so.1.0.0

解决：

1、拷贝并修改/qt-linux/src/deps/openssl/lib/下的 libssl.so文件名称为libssl.so.1.0.0

2、把该文件拷贝到目录/lib/x86_64-linux-gnu/

问题

Error while building/deploying project youduqt (kit: Desktop Qt 5.7.1 GCC 64bit)

When executing step “qmake”

原因：缺少cmake

解决：

sudo apt-get install cmake

还是同一个错误可以重启软件试下

问题

:-1: error: g++：命令未找到

原因：缺少g++

解决

sudo apt-get install g++

还是同一个错误可以重启软件试下

问题

:-1: error: /home/xinda/Qt5.7.1/5.7/gcc_64/bin/qmake：命令未找到

解决

https://blog.csdn.net/GitzLiu/article/details/53993978

问题

error while loading shared libraries: libcrypto.so.1.0.0: cannot open shared object file: No such file or directory

解决

拷贝并修改/qt-linux/src/deps/openssl/lib/下的 libcrypto.so文件名称为libcrypto.so.1.0.0

（打包时依赖）把该文件拷贝到目录/lib/x86_64-linux-gnu/

问题

unknown debugger type “no engine”

原因：缺少gdb

解决：

sudo apt-get install gdb

然后重启qtCreator
