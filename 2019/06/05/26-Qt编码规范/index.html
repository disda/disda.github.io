Qt编码规范

概要
大部分遵循我们原本pc下的代码规范。但是对于现新开展的Qt项目，因为和我们现有的duilib的PC项目还是有区别的。所以有一下几点要求，希望大家进行遵循，没有什么坏处。如果大家还有其他合理的能提高我们代码质量的，可以继续添加。

命名总结

  1. 变量名/函数名采用驼峰命名法(lowerCaseCamel), 首字母缩写词出现的命名中, 缩写也用驼峰命名。

short counter;
char itemDelimiter;
void myXmlStreamReader();

  1. private成员变量添加 m_ 前缀，全局变量添加 g_ 前缀。

  2. private函数以 _ 开头，首字母小写，例：”_funName()”。

  3. public首字母大写，例：“GetEntShortName()”;

  4. slots以 _on 开头。

  5. signals以 on 开头。

  6. 常量命名：常量不含前缀且应该大写，单词间有下划线，包括全局常量和宏定义。

编码规范

  1. 指针/引用 在类型名和*或&之间有一个空格, 在*或&与变量名之间没有空格。

QListWidgetItem *item(int row) const;
int row(const QListWidgetItem *item) const;
void insertItem(int row, QListWidgetItem *item);
void insertItem(int row, const QString &label);
2. 空白行与空格的使用

用空行在适当的地方划分代码块

总是只用一个空行

在关键词和花括号之间总是只用一个空格符

// Wrong
if(foo){
}

// Correct
if (foo) {
}
3. 逗号

逗号左边没有空格，逗号右边有一个空格

#include
#include

int main(int argc, char *argv[]) {
QT_REQUIRE_VERSION(argc, argv, “4.0.2”);

QApplication app(argc, argv);

return app.exec();

}
4. 花括号

使用紧贴括号：左括号和语句的开头在同一行，如果右括号是紧跟在一个关键词之后的，则右括号和该关键词在同一行。

if, while 方法体只有一行语句时，也需要添加花括号。

// Wrong
if (codec)
{
}
else
{
}

// Correct
if (codec) {
} else {
}
代码规范

  1. 类声明中的编排

请按照Q_OBJECT, public, signals, slots, protected, private的顺序将成员变量和函数进行编排。

  1. 对于界面类，如果我们使用了信号和槽，请一定在class声明的第一行加上Q_OBJECT。
  1. 对于信号和槽函数的连接，使用返回值对信号和槽连接是否成功进行判断。

bool bConnected = QObject::connect();Q_ASSERT(bConnected);

  1. Qt下的一些内存问题

QObject及其派生类的对象，如果其parent非0，那么parent析构时会析构该对象。所以我们如果不想delete，尽量将控件的parent设置为其父窗口。
QWidget及其派生类的对象，可以设置Qt::WA_DeleteOnClose标志位，当close会析构该对象。
QAbstractAnimation派生类的对象，可以设置QAbstractAnimation::DeleteWhenStopped。
QRunnable::setAutoDelete()，MediaSource::setAutoDelete()

  1. 界面设计

设计QT界面时，请尽量使用布局管理器进行布局管理，避免使用绝对坐标。

  1. 不推荐直接删除QObject,在带有事件循环的程序中，最好利用QObject::deleteLater()来删除。
